Relay

Overview

The Relay library provides functions for controlling NI switch hardware used in the DU1080 HW4 ATP system.

It supports initialization, relay switching, signal routing, and discuss signal control.

1.4 RelayInitialize

1.4.1 Header

int DLLEXPORT _stdcall RelayInitialize(char *resource_name, ViSession *session, ViBoolean *test_result, char *status_message);

1.4.2 Parameter

resource_name

session

test_result

status_message

1.4.3 Purpose

Initialize the NI switch module and establish a communication session with the switch hardware.

If initialization succeeds, a valid NI Switch session is returned. Otherwise, an error message is generated and stored in status_message.

1.4.4 Return Value

Returns an integer status code.

  • 0 indicates the operation completed successfully.
  • A non-zero value indicates a NI Switch or VISA error occurred during initialization.

1.5 RelaySet

1.5.1 Header

int DLLEXPORT _stdcall RelaySet(ViSession session, char channel[], int operation);

1.5.2 Purpose

Controls the open and closed state of specified relay channel.

The function first checks whether each requested connection path already exists. It disconnect the existing path when necessary and then establishes the path required for the selected operation.

1.5.3 Parameters

  • session An active NI Switch session established by RelayInitialize.
  • channel The relay channel to control. The function uses this value to identify the corresponding common (com), normally closed (nc), and normally
  • operation Specifies the requested relay state:

    • ‘0’ opens the switch by disconnecting the normally open connection and connecting the normally closed connection.

    • Any non-zero value closes the switch by disconnecting the normally closed connection and connecting the normally open connection.

1.5.4 Return Value

Return 0: errors Return -1: Return -2:
