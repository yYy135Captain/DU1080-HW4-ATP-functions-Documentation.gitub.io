MUX

Overview

The MUX module controls the NI Switch hardware used in DU1080 HW4 ATP system. It manages switch initialization, relay control, signal routing, and session cleanup. The module provides a software interface for connecting and disconnecting signal paths required during automated testing.

1.19 MUXInitialize

1.19.1 Header

int DLLEXPORT __stdcall MUXInitialize(char *resource_name, char *topology, ViSession *session, ViBoolean *test_result, char *status_message);

1.19.2 Purpose

Initializes a NI Switch MUX device using the specified resource name and topology. The function returns the initialized switch session, reports whether initialization succeeded, and generates a status message describing the result.

1.19.3 Parameter

Parameter Type Description
resource_name char * resource name.
topology char * topology to be set in switch module.
session ViSession * session.
test_result ViBoolean * test result.
status_message char * status of task message.

1.19.4 Return Value

Error Description
VI_SUCCESS initialized successfully.

1.20.1 Header

int DLLEXPORT __stdcall MUXSet(ViSession session, char *channel_1, char *channel_2, int operation);

1.20.2 Purpose

1.21 MUXClose

1.21.1 Header

int DLLEXPORT __stdcall MUXClose(ViSession session, char *status_message);

1.21.2 Purpose

Resets the NI Switch device and closes the switch session.

1.21.3 Parameter

Parameter Type Description
session ViSession NI Switch Session
status_message char * Status message buffer

1.21.4 Return Value

Error Description
VI_SUCCESS session closed successfully.

1.22 MUXOpenAllRelays

1.22.1 Header

int DLLEXPORT __stdcall MUXOpenAllRelays(ViSession session);

1.22.2 Purpose

Open all relays on the NI Switch module and resets the switch.

1.22.3 Parameter

Parameter Type Description
session ViSession NI Switch Session

1.22.4 Return Value

Error Description
VI_SUCCESS Operation completed successfully.
Other value NI Switch error code.

1.23 MUXSignalSet

1.23.1 Header

int DLLEXPORT __stdcall MUXSignalSet(ViSession session, char *signal_name, void **signal_list, char *status_message);

1.23.2 Purpose

Searches the signal list for the specified signal and executes the corresponding MUX switching operation.

1.23.3 Parameter

Parameter Type Description
session ViSession NI Switch session
signal_name |char *| Signal to execute | |signal_list|void **| Signal list | |status_message|char *` Status message buffer.

1.23.4 Return Value

Error Description
0 Signal executed successfully
-1 Signal list is not loaded
-2 Invalid signal action
-3 Signal not found or MUX operation failed.
-4 Invalid channel number
